2. Chunking

The same doc, , split into the chunks that actually went into the database.

Splitting is two-pass. First the doc is cut on its H2 headers, so a chunk never straddles two sections. Then each section is windowed into fixed-size pieces of 256 tokens with a 32-token overlap. Metadata (source, doc_type, section, char offsets, token count) is attached at chunk time, not later. Short sections become a single chunk; only longer ones like Columns get windowed.

overlap shared with the adjacent chunk in the same section
What the overlap prevents. A fact split mid-sentence at a window boundary would otherwise live in neither chunk's searchable text. The 32-token overlap means any span near a boundary appears in both neighbours, so a query can match it from either side.
